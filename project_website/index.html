Smart Plant IoT System

Smart Plant IoT System is a comprehensive IoT platform featuring gesture recognition, voice interaction, soil moisture monitoring, and a real-time web dashboard for smart plant care.

The system combines ESP32-based hardware, a Flask server, MediaPipe gesture detection, speech processing, and a browser-based dashboard to provide an intuitive and interactive way to monitor and care for indoor plants.

Motivation

Motivation

Busy plant owners often find it difficult to monitor plant health and watering needs on a daily basis. Traditional sensor-based systems provide numbers but lack a natural, human-friendly way to interact with the system.

Our goals are:

  • Enable voice interaction with the plant system through a conversational agent.
  • Provide real-time gesture feedback using a camera and computer vision.
  • Monitor soil moisture continuously and visualize its history.
  • Offer an integrated web dashboard that shows system status in real time.

System

The Smart Plant IoT System consists of two ESP32-based devices and a Flask server running on a computer. The ESP32 Huzzah handles voice interaction, OLED display, and soil moisture sensing, while the ESP32-CAM provides a video stream for gesture recognition. The Flask server coordinates speech processing, gesture recognition, moisture data storage, and the web dashboard.

Architecture

System Architecture Overview

Hardware Components

  • ESP32 Huzzah: Main controller for microphone (SPH0645), speaker (MAX98357A), OLED display (SSD1306), and soil moisture sensor (GPIO 39).
  • ESP32-CAM: Streams video from the OV2640 camera module via :81/stream for gesture recognition.
  • Soil Moisture Sensor: Provides continuous readings of soil moisture level.
  • Network: Both ESP32 boards connect to the same WiFi network as the Flask server.

Hardware Setup

Server and Software

  • Flask Server (server.py): Central hub providing REST APIs and serving the web dashboard.
  • Web Dashboard (/): Shows moisture history chart, gesture status, and conversation controls.
  • STT/TTS API: /api/stt and /api/tts endpoints using Google Speech Recognition and pyttsx3 for text-to-speech, plus POE API (GPT-5) for conversation.
  • Gesture Recognition Thread: Uses OpenCV to capture frames from ESP32-CAM and MediaPipe for hand/pose detection and gesture classification (Hi, Wow, Good).
  • Moisture Data Storage: Maintains latest moisture data in memory with thread-safe access for both APIs and the dashboard.

Results and Current Progress

The current implementation demonstrates an end-to-end pipeline connecting the ESP32 devices and the Flask server. We have:

  • Verified soil moisture sensing and real-time updates on the web dashboard.
  • Established a working video stream from ESP32-CAM and tested gesture detection with MediaPipe.
  • Implemented a voice interaction flow from microphone input to STT, GPT-based reply, and TTS output.
  • Integrated multiple threads and APIs on the server to run concurrently without blocking.
